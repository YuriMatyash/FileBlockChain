# PrintChain Demo Flow

This is the ordered final review demo. It keeps the project local and clearly separates real smart-contract behavior from mock/demo IPFS and x402 behavior.

## Command order

Run the project in this order:

1. `npm install`
2. `npm install --prefix frontend`
3. `npm install --prefix backend`
4. `npx hardhat node`
5. `npm run deploy:local`
6. `npm run seed:local`
7. `npm run backend`
8. `npm run frontend`
9. Connect MetaMask to `localhost` chain `31337`.
10. Run marketplace and x402 demos.

Use separate terminals for the long-running Hardhat node, backend, and frontend.

## Demo accounts

Use local Hardhat accounts printed by `npx hardhat node`:

| Role | Account |
| --- | --- |
| Deployer / PRINT owner | Account #0 |
| Creator / initial seller | Account #1 |
| Buyer | Account #2 |
| Resale buyer | Account #3 |

Only import local Hardhat test keys. Never use these accounts for real funds.

## Part A — Start and inspect the DApp

1. Open the frontend, usually `http://127.0.0.1:5173`.
2. Connect MetaMask on Hardhat chain `31337`.
3. Confirm the app loads the local `contracts.json` generated by deployment.
4. Show the marketplace listing seeded by `npm run seed:local`.
5. Point out that the card represents a manufacturing/use license NFT, not merely ownership of a raw STL file.
6. Show PRINT balance/token information if available for the connected account.

## Part B — Mint/upload flow with mock IPFS

1. Switch MetaMask to the creator account.
2. Open the upload/mint section.
3. Choose or enter a sample manufacturing file reference such as STL, STEP, 3MF, CNC, ZIP, PDF, or a technical drawing.
4. Optionally provide a preview image/render.
5. Enter title, summary, category, file type, documentation, compatibility, and suggested initial ETH price.
6. Submit the form.
7. Explain that default upload mode is mock/demo: the app creates CID-like values locally and stores lightweight metadata in browser session storage.
8. Confirm the mint transaction in MetaMask.
9. Show that the license stores file CID, metadata CID/token URI, creator, mint timestamp, and initial history entry.

## Part C — List and cancel

1. Stay on the owner/creator account.
2. Open **My Licenses**.
3. Enter an ETH price and list the license.
4. Show the active marketplace listing.
5. Optionally cancel the listing to demonstrate delisting.
6. Re-list the license before continuing to the purchase demo.

## Part D — First purchase with ETH

1. Switch MetaMask to buyer account #2.
2. Refresh/load marketplace data.
3. Buy the listed license NFT with the exact ETH price.
4. Confirm the transaction in MetaMask.
5. Show that ownership changed to the buyer.
6. Show the ownership history entry with action `SALE`, price, timestamp, previous owner, and new owner.
7. Explain payment split:
   - 10% royalty is calculated for the original creator/designer.
   - 90% is paid to the current seller.
   - If the first seller is also the creator, both payments go to the same creator address, so the creator effectively receives the full sale price.

## Part E — Resale and royalty demonstration

1. Stay on buyer account #2 after it owns the NFT.
2. List the purchased license for resale.
3. Switch MetaMask to resale buyer account #3.
4. Buy the resale listing.
5. Show ownership changed to account #3.
6. Show history now includes another `SALE` entry.
7. Explain that the original creator still receives the 10% royalty on resale, while the reseller receives 90%.

## Part F — Details, history, and timestamps

On the NFT details view, show:

- Token ID.
- Title and description.
- Creator/designer address.
- Current owner.
- File CID.
- Metadata CID/token URI.
- Mint timestamp.
- Ownership history.
- Price values stored with history entries.
- Current listing status.
- Royalty explanation.

## Part G — x402 / HTTP 402 mock demo

The x402 route is intentionally mocked. It demonstrates HTTP `402 Payment Required` behavior but does not perform real x402 settlement.

### Browser/UI demo

1. Open a license details view.
2. Click the unpaid preview/demo button if available.
3. Show the HTTP 402-style response/error.
4. Click the mock-paid/demo authorization button.
5. Show protected preview JSON/data.

### Terminal demo

Unpaid request, expected HTTP `402`:

```bash
curl -i http://127.0.0.1:4000/api/paid-preview/1
```

Mock-paid request, expected HTTP `200`:

```bash
curl -i -H "x-printchain-demo-payment: paid" http://127.0.0.1:4000/api/paid-preview/1
```

## What to say during review

- PrintChain is a DApp marketplace for digital manufacturing license NFTs.
- Each NFT represents a license to use, print, or manufacture the digital model/file.
- PRINT is a reward token, not the purchase currency.
- Marketplace purchases use local test ETH.
- The marketplace enforces 10% creator/designer royalty on sales.
- Ownership history, price values, and timestamps are kept on-chain.
- Files are not stored on-chain.
- IPFS upload is mock/demo by default, with CID-like metadata for local review.
- x402 is a mock/demo HTTP 402 route, not real settlement.
